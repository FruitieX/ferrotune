# Android Emulator Smoke Testing

This project has a Playwright-powered Android emulator smoke test for the Tauri app's queue playback path.

## Prerequisites

- Run Android tooling inside the Android nix shell:

```bash
nix develop .#android
```

- Start an Android emulator and confirm that ADB can see it:

```bash
adb devices
```

- The smoke test expects the debug Tauri APK path generated by the normal Android deploy flow:

```text
client/src-tauri/gen/android/app/build/outputs/apk/universal/debug/app-universal-debug.apk
```

## One-command smoke run

From the repository root:

```bash
bash scripts/android-emulator-smoke.sh
```

That wrapper will:

1. wait for an attached emulator/device
2. rebuild and reinstall the debug Android app
3. run the Playwright Android smoke spec via Moon

## Direct Moon task

Invoke the Moon task directly:

```bash
moon run client:test-android-emulator
```

That task always rebuilds and reinstalls the debug Android app before running Playwright, so it does not rely on whatever APK is already present on the device.

## Device selection

- Use `ANDROID_SERIAL` to target a specific emulator/device when multiple are attached.
- Use `ANDROID_ADB_SERVER_ADDRESS` if your ADB server is not on `127.0.0.1`.

Example:

```bash
ANDROID_SERIAL=emulator-5554 bash scripts/android-emulator-smoke.sh
```

## What the smoke test covers

- installs the current debug APK
- clears app storage for a clean launch
- connects the app to a temporary fixture-backed Ferrotune server
- starts album playback
- opens the queue sheet
- presses the queue item play control for a later track
- asserts both UI state and native Android logcat `playAtIndex` logs

## Notes

- The Playwright Android API is experimental and requires ADB.
- The smoke test is intentionally separate from the default `pre-ci` flow because it needs a running emulator.
